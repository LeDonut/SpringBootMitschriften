### Hilfsklassen im Spring Framework

Das Framework von Sprint enthält ca. 400 Pakete mit rund 4000 Typendeklarationen. Die [Javadoc](https://docs.spring.io/spring-framework/docs/current/javadoc-api/index.html) gibt eine Übersicht.  

Aus folgenden Teilen setzt sich das Spring Framework zusammen:
- [spring-aop](https://www.baeldung.com/spring-aop)
- [spring-aspects](https://dev.to/jarjanazy/intro-to-spring-s-aspects-4pee)
- [spring-beans](https://www.baeldung.com/spring-bean)
- [spring-context](https://www.baeldung.com/spring-application-context)
- spring-context-indexer
- spring-context-support
- [spring-core](https://docs.spring.io/spring-framework/reference/core.html)
- spring-core-graalvm
- spring-core-test
- spring-expression
- spring-instrument
- spring-jcl
- [spring-jdbc](https://www.baeldung.com/spring-jdbc-jdbctemplate)
- spring-jms
- spring-messaging
- spring-orm
- spring-oxm
- spring-r2dbc
- [spring-test](https://www.baeldung.com/spring-boot-testing)
- spring-tx
- spring-web
- spring-webflux
- spring-webmvc
- spring-websocket
- framework-bom
- integration-tests

Im Gegensatz gibt es bei JakartaEE zu Spring, keinen direkten Unterschied zwischen "Container" und "Implementierung", da diese zusammen eine Einheit bilden.
